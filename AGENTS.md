# AGENTS.md — LetsLearnOS Project Conventions

Canonical conventions for AI agents and human contributors working in this
repository. Read `GOAL.md` before changing product scope.

## Source of truth

This repository is the only active LetsLearnOS source tree. `GOAL.md` defines
the deliverable and release gate. `DECISIONS.md` is the append-only architecture
decision log; add dated decisions when architecture changes.

If a request conflicts with `GOAL.md`, surface the conflict and obtain owner
approval before updating the goal and implementation together.

## Stack

- **Monorepo:** pnpm workspaces, with packages under `artifacts/*` and `lib/*`.
- **Frontend:** React 19, Vite, TypeScript, Tailwind v4, wouter, framer-motion,
  and an offline service worker (`artifacts/letslearnos`).
- **Backend:** Node, Express 5, TypeScript, and esbuild
  (`artifacts/api-server`).
- **Database:** Drizzle ORM; Postgres in development and SQLite on the kiosk.
- **Target OS:** Ubuntu LTS with Chromium kiosk on x86_64 hardware.

## Commands

```bash
pnpm install
pnpm run typecheck
pnpm --filter @workspace/letslearnos build
pnpm --filter @workspace/api-server test
pnpm --filter @workspace/api-server build
python3 scripts/generate-asset-manifest.py
python3 scripts/verify-assets.py --repository
```

Native esbuild, Rollup, Lightning CSS, and SQLite packages are pinned for the
Linux x64 release environment. Use CI for authoritative builds on other host
architectures.

## Coding conventions

- Touch targets are at least 88×88 px; no critical UI belongs in the top 80 px.
- Use positive-only feedback for the younger learner profile.
- Never put API keys or backend environment values in frontend code.
- OpenAI calls originate only from the backend and must be optional.
- Offline browser narration remains available when OpenAI is unconfigured.
- Runtime character art resolves locally and falls back to the bundled neutral
  SVG. Do not call external character APIs from the running app.
- The public repository contains no proprietary artwork or commercial music.
- Math is template-based and never generated by an AI model.
- Keep Postgres and SQLite schemas structurally identical.
- SQLite initialization is idempotent; destructive migrations require explicit
  owner approval.
- Keep `pnpm run typecheck` green and add tests under the relevant `tests/`
  directory.

## OpenAI configuration

- `OPENAI_API_KEY` or `OPENAI_ACCESS_TOKEN` is read by the backend only and is
  always optional.
- `.env` files and real credentials are never committed.
- Configuration examples contain blank placeholders only.
- Do not log credentials or include them in errors, URLs, browser storage, or
  database rows.
- Narration sent to OpenAI must come from static or template-generated learning
  content; do not send child names, progress records, PINs, or other profile
  data.
- Clearly disclose that enabled OpenAI narration uses an AI-generated voice.

## Safety rails

- The `kids` user never receives sudo, a shell, or a desktop.
- Parent PINs are stored as salted hashes, never plaintext or environment
  variables.
- Code that writes to `/etc`, `/opt`, or installs packages is produced as a
  human-run script. Contributors do not execute those scripts.
- Never commit `.env`, secrets, private infrastructure addresses, or personal
  profile data.
- Preserve unrelated user work in a dirty worktree.

## Public-release audit

Before publishing, verify all of the following:

```bash
scripts/audit-public-release.sh
pnpm run typecheck
pnpm --filter @workspace/api-server test
pnpm --filter @workspace/letslearnos build
pnpm --filter @workspace/api-server build
```

All reference searches must return no matches. The example environment file may
name `OPENAI_API_KEY`, but its value must remain blank.

## Handoff protocol

Report: files changed, one verification command or short procedure, remaining
work, and any architectural decision that belongs in `DECISIONS.md`.
